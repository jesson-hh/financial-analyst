## 0. 一句话

### 3.2 环节层(7 群 = 28 个满信号环节 + 2 个相邻链接口)

计数口径:满信号环节 28 个(A1-A6、B1-B5、C1-C5、I1-I4、M1-M2、F1-F4、G1-G2);相邻链接口 2 个(G3 机器人、G4 智能驾驶,只挂链接不建信号,不计入 28)。

粒度原则:**按"超额发生地"切**(光模块/CPO 分开、PCB 与其上游材料分开、液冷标注整机与零部件)。每环节字段:`id/name/group/logic/keywords/ths_concepts/stocks(锚票)/global(全球坐标)`。

**群 A 芯片与材料**:A1 国产AI芯片(寒武纪/海光/摩尔线程/沐曦)、A2 先进制程代工(中芯/华虹)、A3 半导体设备(北方华创/中微/长川)、A4 先进封装 CoWoS(通富/长电/华天)、A5 存储/HBM(江波龙/兆易/佰维/澜起/香农)、A6 半导体与PCB上游材料(中船特气/江丰/东材/生益)。

**群 B 服务器与配套**:B1 AI服务器/ODM(工业富联/浪潮/中科曙光/华勤)、B2 高速PCB(沪电/胜宏/深南/鹏鼎)、B3 服务器电源/HVDC(麦格米特/中恒/新雷能/欧陆通)、B4 液冷·整机+UQD/CDU零部件(英维克/申菱/高澜/川环)、B5 铜连接/高速连接器(立讯/华丰/中航光电;沃尔核材待核)。

**群 C 光与网络**:C1 光模块·可插拔 800G/1.6T(中际旭创/新易盛/华工)、C2 光芯片/光器件(源杰/仕佳/长光华芯/天孚/太辰光)、C3 CPO/硅光(旭创/新易盛/天孚/罗博特科)、C4 OCS 全光交换(光库/三安)、C5 交换机/网络设备(锐捷/紫光股份/盛科)。

**群 I 算力基建**:I1 IDC/AIDC(润泽/光环新网/数据港)、I2 算力租赁/智算服务(首都在线/协创/润泽)、I3 算电协同/电力设备(白云电器等,票池待概念补全)、I4 运营商/云平台(移动/电信/联通)。

**群 M 模型与数据**:M1 基础大模型/MaaS(讯飞/昆仑/360;生态映射 DeepSeek链/豆包链/千问链)、M2 数据要素/语料(海天瑞声/拓尔思)。

> ID 约定:环节 = 群字母+序号(A/B/C/I/M/F/G,刻意避开驱动前缀 D 与传导边前缀 T);驱动 = D1-D7;传导边 = T1-T15;叙事 = N1-N8。`F*` 表示 F1-F4 全体。

**群 F 应用**:F1 AI办公/Agent(金山办公/用友/赛意)、F2 AI金融(同花顺/恒生/东财)、F3 AIGC/传媒游戏(昆仑/万兴/巨人)、F4 AI垂直行业·医疗教育工业(卫宁/讯飞/鼎捷)。

**群 G 端侧与具身**:G1 AI手机/PC/端侧SoC(立讯/华勤/恒玄/瑞芯微)、G2 AI眼镜/XR(歌尔/弘景/视涯)、G3 机器人/具身智能(**相邻链**,只建接口环节挂链接不建满信号)、G4 智能驾驶(**相邻链**,同上)。

票池生成规则:锚票人工定(上表),完整票池 = `同花顺概念成分 ∩ (研报提及 ∪ 行业分类)` 自动生成候选再人工修;`keywords` 供 LLM 正文匹配(吸取"铜连接标题层缺席"教训)。

### 3.3 全球坐标(环节层第六字段组 + 股票逻辑类型学)

每环节强制回答:国际谁主导 → 国内什么地位(领先/并跑/追赶/短板)→ 壁垒在哪 → 股票逻辑哪一型。

**股票逻辑五型**(LLM 读研报也按此打标签):

| 型 | 定义 | 跟什么信号 |
|---|---|---|
| β 全球需求型 | 国内产能/份额全球领先,吃海外 capex | 北美capex、订单排产、出货量 |
| Ω 国产替代期权型 | 国内短板+管制倒逼,炒突破预期 | 政策/测评名单、突破事件、大厂订单 |
| Δ 涨价周期型 | 供需缺口驱动价格 | 合约价环比、库存、产能利用率 |
| Θ 技术路线期权型 | 新路线(CPO/800V/超节点)重估 | 定调事件、量产验证节点 |
| Ψ 映射主题型 | A股无纯标的,靠海外/生态映射 | 海外产品节奏、开源发布 |

环节可主+次两型;**Δ+Ω 双击型 = 十倍股发源地**(如 C2 光芯片)。

28 环节全球坐标定位(v1 初值,研报会持续修正):A1 短板→追赶(Ω→兑现过渡)、A2 短板(Ω+产能β)、A3 追赶★(Ω兑现)、A4 并跑(β+Ω)、A5 短板但模组吃涨价(Δ主+Ω)、A6 部分领先★(Δ+扩产β)、B1 产能领先★(β兑现)、B2 全球领先★★(β最硬)、B3 并跑→领先★(β+Θ)、B4 领先★(渗透率β)、B5 追赶(Θ)、C1 全球绝对领先★★(β,盯CPO替代)、C2 追赶突破★★(Δ+Ω双击)、C3 跟随(Θ高分歧)、C4 代工独占(Ψ→β)、C5 白牌强芯片追赶(β+Ω)、I1 国内市场★(量价β)、I2 竞争加剧(推理β)、I3 政策新基建、I4 红利+重估、M1 映射(Ψ)、M2 主题、F1 先兑现★(业绩验证)、F2 业绩验证、F3/F4 主题轮动(Ψ)、G1 制造主导(新品β)、G2 供应链主导★(新品β)。

**"好节点"高亮规则**:`国内地位∈{领先,并跑}` × `逻辑含β或Δ` × `需求加速`。当前第一梯队:PCB、光模块、光芯片(双击)、上游材料、液冷、电源HVDC、服务器ODM、半导体设备;期权梯队:国产AI芯片、HBM国产链、CPO、铜连接。

研报讨论"国产化率/份额/技术差距/认证进展"时抽成 `global_update` 信号,持续修正坐标——**坐标是活数据**。

### 3.6 信号定义

量化侧(每环节票池聚合,现算+缓存,全读现有产物):

| 信号 | 算法 | 数据源 |
|---|---|---|
| 行情温度 | 票池等权 20 日动量、相对全A等权超额 | qlib 日线 + `eqw_market` |
| 拥挤度 | 环节成交额占全A比 + 20日变化 | qlib amount |
| 资金流 | 主力净流入 5/20 日 | `stock_fund_flow_daily` |
| 模型视角 | 票池 v4 分位均值 | `v4_ranking_latest` |
| 广度 | 涨跌家数比、创新高家数 | qlib 日线 |

文本侧(DeepSeek 每篇研报一条抽取记录,全带 doc_id):

  "doc_id": "...", "title": "...", "org": "...", "publish_ts": "...",

  "catalysts":  [{"type": "订单|涨价|扩产|技术突破|政策|业绩|认证|新品", "desc": "...", "date_hint": "..."}],
  "edges":      [{"edge_id": "E1", "verdict": "支持|否证", "evidence": "..."}],
  "narratives": [{"narrative_id": "N2", "stance": "多|中|空"}],
  "global_updates": [{"segment_id": "A1", "field": "国产化率|份额|技术差距|认证", "content": "..."}],

聚合呈现(**诚实原则:不做黑盒合成分**):每环节双轴四象限——横轴行情温度 × 纵轴研报景气(近 30 日立场,新鲜度指数衰减半衰期 7 天 × 强度 1-3 加权,半衰期常量可配置),外加分歧度(立场方差)。四象限:双热 / 研报热行情冷(潜伏)/ 行情热研报冷(情绪)/ 双冷。传导边显示最新验证指标 + 近 30 日支持/否证计数。

### 4.2 ingest 管线(手动触发,增量)

1. `POST /industry/ingest` → 单飞锁(temp 锁文件,仿 regen `_acquire_regen_lock`)→ 后台线程执行,立即返回受理凭证
2. 扫 `G:\stocks\stock_data\text_source\documents.parquet`:增量 = `publish_ts >= 水位` + 已抽取 doc_id 剔重(2026-07-03 终审 fast-follow #5:>= 防同日晚回填漏扫,剔重防重复 LLM 花费/totals 双计,剔重先于 limit);筛选 = `doc_type=industry_research` 全收 + `company_research` 命中(票池 ∪ keywords 正文匹配);跨仓路径经 env `GL_TEXT_SOURCE_ROOT` 可覆盖(仿 `GL_F10_ROOT` 先例)
3. 每篇:读 text_path 全文(过长截断分段,首版单篇上限约 20k 字符,超出取头+尾)→ DeepSeek 抽取(见 4.3)→ 校验环节/边/叙事 id 合法性(非法 id 丢弃该条目并记 warning)→ append `extractions.jsonl`
4. 全部成功才推水位;失败篇目记入 state(`failed_docs`),不阻断其余
5. `GET /industry/ingest_state`:进度/水位/失败清单/累计 token 用量显形

### 4.3 LLM 调用约定(照 `screen/llm.py` 模式)

- 复用引擎 `LLMClient`,显式 `config_path=仓内 config/llm.yaml`,默认 `deepseek-chat` + `response_format=json_object`
- `asyncio.wait_for` 超时 90s/篇(研报全文比选股 prompt 长);`asyncio.Semaphore(3)` 限并发;协程内同步 I/O 一律 `to_thread`(9999 看门狗红线)
- 系统提示禁编造:只能引用原文中出现的信息,`quote` 必须是原文子串(实现做包含校验,失败降级为无 quote 并标注)
- 真失败返回 `{ok:false, reason}` 记入 failed_docs,绝不伪造抽取结果
- 212 篇 parse_failed(扫描版 PDF,text_chars=0)直接跳过并计数显形

### 4.4 API

| 端点 | 作用 |
|---|---|
| `GET /industry/board` | 全量看板 JSON:驱动+环节(含信号)+边(含验证状态)+叙事(含温度)+新鲜度元数据;服务端缓存 TTL 10min,`?refresh=1` 强刷 |
| `POST /industry/ingest` | 手动触发增量批处理(单飞) |
| `GET /industry/ingest_state` | 批处理状态/水位/失败清单/token 计量 |
| `GET /industry/segment/{id}` | 环节明细:研报观点流(带引句)+个股表(行情/资金/v4 分位) |
| `GET /industry/doc/{doc_id}` | 单篇溯源:该研报的完整抽取记录 + 原文路径 |

全部失败诚实:`ok:false + reason`,HTTP 200(仓内惯例)。

### 4.6 错误处理与诚实红线汇总

1. 跨仓路径不存在/parquet 损坏 → board 返回 `ok:false + reason`,前端显示断供徽章(chunks 表有 corrupt 历史,读取层 try/except)
2. LLM 失败/超时 → 该篇记 failed_docs,可重试;绝不冒充
3. 引句校验:quote 非原文子串 → 降级为无引句并标注
4. 框架 YAML 校验失败(坏 id 引用)→ 模块加载即报,fail fast
5. 行情产物缺失(v4/资金流/eqw)→ 对应信号显示 null + 原因,不静默补零
