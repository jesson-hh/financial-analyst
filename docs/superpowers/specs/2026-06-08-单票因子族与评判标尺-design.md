# 单票因子族 + 评判标尺 — 设计文档

- 日期: 2026-06-08
- 状态: 已与用户确认设计,进入 Phase 1 实现
- 范围: 为观澜工作流平台新增"单票/事件型"因子族,并补齐对应的正确评判标准

## 1. 背景与动机

平台现有因子体系是为**横截面(cross-section)选股**优化的:评判用 RankIC/ICIR(把当日全体股票按因子排序,与前向收益排序求相关)。

用户提出要做**单只股票**视角的因子:筹码分布、跟随属性、共振、反弹、消息面、情绪。这些在单票上**横截面 IC 无定义/退化**(n=1 无法排序)。因此本工程要做两件事:
1. 新增这些**因子族**;
2. 补齐它们对应的、学界认可的**评判标准**。

## 2. 核心架构 — 两类因子,两条评判线

```
连续型因子(筹码状态 / 共振级别 / 跟随强度 / 情绪强度) ─→ 时序IC (tsic 节点,已通)
离散型因子(反弹 / 消息面事件 / 情绪极值 / 龙虎榜)      ─→ 事件研究CAR (engine/eval/event.py,待接节点)
```

不发明新标尺,而是**把引擎里已写好的 `build_event_report` 接出来** + 复用已通的 `tsic`。

### 三层架构(界定"引擎改动")
| 层 | 位置 | P1 是否改动 |
|---|---|---|
| ① 引擎(因子计算核心) | `engine/financial_analyst/` — 字段 panel.py / 算子 operators.py / DSL白名单 expr.py / 加载 qlib_binary.py / 评估器 eval/*.py | **不动** |
| ② 薄壳后端 | `guanlan_v2/workflow/api.py`(调用引擎,FastAPI 路由) | 动(加路由) |
| ③ 前端 | `ui/factor/workflow.jsx` | 动(加节点) |

**决策(用户确认):P1 零引擎改动。** CAR 用引擎现成的"市场均值调整异常收益"(一阶近似),并在 UI 老实标注;事件研究的严谨度升级(beta 调整异常收益 + Newey-West t 值)推迟到 **P2**,届时随 `regbeta` 解锁一并做。

## 3. Phase 1 设计(零引擎改动,先出真东西)

### 3.0 事件研究的本质约束(读 event.py 后确认)
`build_event_report(panel, compute, config, factor_label, horizons)` 的市场调整异常收益 `_excess()` = 逐日减 **面板内等权全市场均值**。推论:
- **事件研究天生是"股票池"操作**:把触发表达式跑遍整个池,收集所有 firing 的 (date,code),聚合算 CAR。
- **单票退化**:池里只有 1 只时,市场=它自己 → `mean_excess≡0`、`t_stat=NaN`。所以 event 节点默认跑**股票池**(csi300 等);单票时只看原始收益 `mean_ret`,并给退化警告。真正的"对 idx_ret 做市场调整"的单票事件研究属 P2 引擎改动。

### 3.1 后端:事件研究节点(薄壳)
`guanlan_v2/workflow/api.py`:
- `FactorEventIn(ModelTrainIn)`:`trigger`(>0 的 DSL 触发表达式)、`horizons=[1,5,10,20]`、`direction`(看多/看空/自动)。
- `_factor_event(body)`:解析 codes(默认池;支持自选)→ 加载 panel → `compile_factor(trigger)` → 构造 `EvalConfig` → 调用引擎 `build_event_report(panel, compute, cfg, label, horizons)` → 输出 `{ok, method:'event', n_events, event_rate, horizons:[{h,n,mean_ret,mean_excess,win_rate,t_stat}], car_curve:[[d,car]], by_year:[[y,n,mean]], expectancy, profit_factor, _label, _universe, status, warnings}`。
- 路由 `POST /factor/event`(实现时确认未被 buddy app `build_app` 遮蔽;若遮蔽退 `/workflow/event`)。
- **诚实护栏**(沿用 + 扩展引擎自带 warnings):`event_rate>50%`→"更像连续因子,请走因子分析/tsic";`n_events<30`→样本不足;`n_codes==1`→单票市场调整退化,看 mean_ret;标注"异常收益=市场均值调整(一阶近似),非 beta 模型;重叠事件 t 值未做 Newey-West"。
- **补**盈亏比/期望值(在节点层从 firing 事件的原始前向收益算:`profit_factor=Σ正/|Σ负|`、`expectancy=P(赢)·均盈 − P(输)·均亏`)——全平台目前缺。

### 3.3 新因子(纯 DSL,用已有算子)
进 `_FACTOR_CATALOG` + `SYSTEM_PROMPT` 配方:
- **反弹(离散→event)**:
  - 距均线乖离反弹:`cross(close, sma(close,5)) * (((delay(close,1)-sma(close,20))/sma(close,20)) <= -0.08)`
  - 连跌后反转:`(returns>0) * (delay(ts_sum((returns<0)*1.0,4),1) >= 4)`
  - RSI14 超卖回升(RSI 用 `sma/max_pair/delta` **重构**,不焊字段)
  - PEAD 代理:`(np_yoy>0.30)*(delta(np_yoy,1)!=0)*(((close-sma(close,20))/sma(close,20))<=-0.05)`
- **情绪·连续(→tsic)**:
  - 换手突增:`turnover_rate / ts_mean(turnover_rate,20)`
  - 振幅强度:`((high-low)/delay(close,1)) / ts_mean((high-low)/delay(close,1),20)`
  - 量价情绪共振:`sign(delta(close,1)) * (turnover_rate / ts_mean(turnover_rate,20))`
- **消息面·价格代理(离散→event)**:
  - 3σ 异动:`abs_((returns - ts_mean(returns,60))/stddev(returns,60)) > 3`
  - 跳空:`abs_(open/delay(close,1) - 1) > 0.05`
  - 放量+方向:`(volume/ts_mean(volume,20) > 3) * (abs_(returns) > 0.03)`

### 3.4 评估路由(教 LLM 对号入座)
SYSTEM_PROMPT 新增:配方 F(离散触发 → 挂 event 节点)、配方 G(连续情绪 → 挂 tsic)。`_autowire_source` 扩展到 event 节点(source.data→event.src)。

## 4. 后续路线(P2 / P3 / P4)

- **P2(小解锁,一次重启)**:expr.py 放开 `regbeta/regresi/rsqr/sequence` →
  - **事件研究严谨度升级**:event.py `_excess` 改 beta 调整(市场模型,对 idx_ret 估 β)+ Newey-West t 值(P1 推迟来的那块)。
  - **共振**族(对大盘/行业/龙头 相关+beta+下行beta)、**跟随属性**族(滞后互相关/方向代理/cov-var beta)。
  - 新增 **rsqr 拟合稳定度评估节点**;给 **tsic 补 t值 + 多滞后衰减剖面**。
  - 建 **GBK-aware parquet→面板事件 loader**,接龙虎榜触发(仅 ~60 天历史 → 只做向前实盘信号,不吹回测)。
- **P3(引擎 fork,最高价值)**:实现 **CYQ 递归 primitive**(`cyq.py` + panel.py @property:获利盘/集中度/成本乖离/ASR),复刻 W1b 模式;NaN 烧入期、复权一致价、优先比率型输出、扩散系数 A 作参数;先 winner_ratio+集中度,tsic 验证。核实并接 `northbound_holding`(个股北向)。
- **P4(独立阶段,用户选"文本也要")**:新闻采集 + NLP 情绪打分管线 → news-polarity / news-volume 字段。排在 P3 之后。

## 5. 评判标尺总表

| 因子形态 | 标尺 | 状态 |
|---|---|---|
| 连续单票(筹码/共振级别/情绪强度) | 时序IC tsic(P2 加 t值+衰减) | 已通 |
| 离散事件/触发(反弹/消息面/龙虎榜) | 事件研究 CAR + 命中率 + t值 + 逐年 + 盈亏比/期望值 | P1 接出 |
| 关系(共振/跟随) | tsic on beta级别&变化 + rsqr 拟合稳定度 | P2 |
| 过拟合控制 | OOS / walk-forward / CSCV-PBO(已有)套到新因子 | 已有 |

## 6. 已焊进设计的诚实约束
- 真实指数只有沪深300(399300.SZ);其它宽基/行业用 `csmean/indmean` 等权代理。
- 文本情绪/消息面 polarity 在 P4 前**做不了**(无打分语料)→ 价格代理顶上。
- 龙虎榜仅 ~60 天历史 → 只做向前实盘信号,不做回测背书。
- "盈余惊喜"= np_yoy 的 YoY 代理,非分析师一致预期。
- P1 的事件研究 CAR = 市场均值调整(一阶近似),beta 调整 + NW 在 P2;单票市场调整退化。
