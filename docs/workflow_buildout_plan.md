## 3. 架构 = 三层基础设施(P0 先搭)

**① 前端:节点注册表 + 通用 DAG 执行器**(把现在硬写的 `run()` 升级成框架)
- 每节点 = `{ spec(已有 SPECS), exec(上游输入,参数,ctx)→输出, render(结果) }`。
- 通用执行器:拓扑排序 → 逐节点 `await exec` → 沿边传类型化数据(dt)→ 节点态 running/done/error → 终端结果进结果抽屉。
- **对标** panda `@work_node`(其 input_model/output_model/run = 我们的 spec/exec/render)。
- **加新节点 = 注册一个三件套**(这就是「后续从网上找信息往里填」的地基)。界面零改动。

**② 后端:在仓内按节点加端点**(`guanlan_v2/<module>/api.py` + `include_router`,cards/seats 先例;import 引擎 primitive 计算、数据走 `get_data_paths`;**不改 `engine/` 副本、不碰 fa-watch-wt**)。对标 panda 各 node `.run()`。

**③ 因子库**:见 §2。
