# 金融助手 (financial-analyst buddy) — 功能全景 · 面向设计

> 这份文档给设计师/设计 agent 用. 目标: 看完能知道这个产品**承载什么能力**、
> **有哪些交互状态**、**展示哪些信息形态**, 从而针对性设计界面 (无论是美化现有
> 终端 TUI, 还是重做成 Web / 桌面 app).
>
> 版本: v1.8.3 · 形态: 终端对话式 TUI (Python + prompt_toolkit) · 语言: 中文为主

---

## 1. 一句话定位

**A 股单股深度研究的对话式工作台.** 用户用自然语言提问 ("看下茅台怎么样" /
"今天主力买什么" / "茅台跌破1200提醒我"), 一个 LLM agent 自动选择并调用底层
26 个工具 (行情/新闻/资金流/研报/盯盘…), 执行后用中文总结. 形态像 Claude Code,
但领域是 A 股投研.

---

## 2. 核心交互范式

```
用户自然语言输入
   ↓
LLM (看到 26 个工具的 schema) 决定调用哪个/哪几个工具
   ↓
运行时执行工具 (本地 DB 查询 / 浏览器抓取 / 子进程跑研报)
   ↓
工具结果回灌进 LLM 上下文
   ↓
LLM 继续调工具 或 输出中文总结
   ↓
渲染进对话流 (transcript)
```

- **多轮工具链**: 一个问题可能触发多个工具顺序调用 (最多 15 轮).
- **流式呈现**: 每个工具调用 / 结果 / LLM 文本逐条 append 进对话流.
- **斜杠命令**: `/` 开头是系统命令 (不走 LLM), 见 §7.

---

## 3. 当前屏幕布局 (现状参考)

终端全屏, 从上到下分区 (大部分是按需出现的条):

```
┌────────────────────────────────────────────────────────┐
│                                                          │
│  对话流 transcript (可滚动, 占满主区)                     │
│  ❯ 用户提问                                              │
│  ▶ 工具调用 tool_name({args})                            │
│  ✓ 工具结果 (截断显示, 完整给 LLM)                        │
│  LLM 中文总结 (markdown 渲染)                            │
│  🔔 盯盘提醒 (后台弹入)                                   │
│                                                          │
├────────────────────────────────────────────────────────┤
│  ⠋ 调用 ths_fund_flow…  ▇▃▄▇▂ +0.8%   ← spinner (思考时) │
├────────────────────────────────────────────────────────┤
│  ⏳ 排队中 → 顺便看比亚迪      ← 队列指示器 (有排队时)      │
├────────────────────────────────────────────────────────┤
│  ⚠ 等待工具确认 run_report — y/n/a/ESC  ← 确认条 (等y/n时)│
├────────────────────────────────────────────────────────┤
│  📜 浏览历史 按 End 回到最新   ← 历史浏览提示 (翻历史时)    │
├────────────────────────────────────────────────────────┤
│  🛡 default · qwen3.5-plus · 🪙 2.3k tok · 👁 盯盘5m(交易中)│ ← 状态行 (常驻)
├────────────────────────────────────────────────────────┤
│  ❯ 用户在此输入█                          ← 输入框 (常驻) │
└────────────────────────────────────────────────────────┘
```

中间 4 条 (spinner / 队列 / 确认 / 历史) 是**按需出现**的状态条; 状态行和输入框常驻.

## 4. 交互状态机 (设计重点 — 每个状态都要能被用户一眼识别)

| 状态 | 触发 | 当前视觉 | 用户能做什么 |
|---|---|---|---|
| **空闲 idle** | 无任务 | 只有状态行+输入框 | 输入提问 / 斜杠命令 |
| **思考中 thinking** | LLM 推理 | spinner 跳动 (K线动画) | 继续打字 (排队) / ESC 取消 |
| **工具执行 tool-running** | 调工具 | spinner 显示工具名 | ESC 取消 / 继续打字 |
| **等待确认 confirm** | 危险工具待批准 | 红色确认条 ⚠ | 输入 y/n/a / ESC |
| **排队 queued** | 思考时又输入 | 黄色 ⏳ 排队条 | ESC 清空队列 |
| **浏览历史 browsing** | 翻看旧对话 | 📜 历史提示 | PageUp/Down / End 回最新 |
| **盯盘中 watching** | /watch on | 状态行 👁 | 后台静默, 触发时弹 🔔 |
| **完成 done** | 一轮结束 | ✓完成 / ⚠警告标记 | 看结果 / 下一问 |

**关键设计点**: 这是个**异步、可打断、可后台**的系统. 用户在 agent 思考时仍能
打字、翻历史、取消. 设计要让"系统在忙 / 在等我 / 在后台盯盘"这几个状态泾渭分明.

## 5. 全部 26 个工具 (按用户意图分组)

工具有 3 档耗时标签: `instant` (本地秒回) / `seconds` (网络抓取 30-120s) /
`minutes` (跑模型 5-8 分钟, 需确认).

### 5.1 速览与行情 (最高频)
| 工具 | 耗时 | 用户问法 |
|---|---|---|
| **stock_brief** | seconds | "看下茅台怎么样" — 一次返回 行情+行业+产业链+新闻+情绪+资金流+上次研报 (核心入口) |
| **quote_lookup** | instant | "茅台 PE/市值多少" (日线 EOD) |
| **realtime_quote** | seconds | "茅台现在多少钱" (盘中实时价/盘口) |

### 5.2 盯盘提醒 (实时性高)
| 工具 | 耗时 | 用户问法 |
|---|---|---|
| **alert_add** | instant | "茅台跌破1200提醒我" (price_below/above, pct_above/below) |
| **alert_list** | instant | "我设了哪些提醒" |
| **alert_remove** | instant | "取消茅台的提醒" |

### 5.3 资金流 / 主力行为 (同花顺)
| 工具 | 耗时 | 用户问法 |
|---|---|---|
| **ths_fund_flow** | seconds | "今天主力买什么"(个股) / "概念主线在哪"(概念) / "行业涨幅"(行业) / "盘中大单"(大单) — 一个工具 4 个 target |
| **fund_flow_change** | instant | "茅台主力是加仓还是出逃" (跨日对比) |
| **iwencai_search** | seconds | "PE<20且ROE>15%的科技股" (问财自然语言选股) |
| **ths_concept_board** | seconds | "最新概念发布" |

### 5.4 新闻 / 情绪
| 工具 | 耗时 | 用户问法 |
|---|---|---|
| **news_query** | instant | "茅台最近有什么新闻" (本地 FTS5 全文检索) |
| **news_collect** | seconds | 刷新新闻 (东方财富/新浪/雪球/同花顺多源) |

### 5.5 研究 / 知识
| 工具 | 耗时 | 用户问法 |
|---|---|---|
| **run_report** | **minutes** | "跑一份茅台深度研报" (5-8min, 出星级评级/目标价/止损/多空论证, 需确认) |
| **chain_for** | instant | "茅台产业链/上下游/同行" |
| **stocks_show** | instant | "之前怎么看茅台/上次评级" (历史研究时间线) |
| **industry_show** | instant | "603259 什么行业" (申万分类) |
| **mainline_radar** | seconds | "这个月主线/板块轮动" (月级状态分类) |
| **morning_brief** | seconds | 盘前晨会简报 (全市场扫描) |
| **dream_review** | instant | "反思/教训" (introspection 记忆提案) |

### 5.6 因子 / 量化 (进阶)
| 工具 | 耗时 | 用户问法 |
|---|---|---|
| **alpha_bench** | **minutes** | 跑 442 因子基准 (需确认) |
| **alpha_snapshot** | seconds | 刷新 top-N 因子快照 |
| **alpha_list** | instant | 列出因子库 (alpha101/gtja191/qlib158) |
| **alpha_show** | instant | 看某因子公式+论文出处 |

### 5.7 个人账户 (雪球/蛋卷)
| 工具 | 耗时 | 用户问法 |
|---|---|---|
| **watchlist_show** | instant | "我的自选股" |
| **fund_snapshot** | instant | "我的蛋卷基金账户总览" |
| **fund_holdings** | instant | "我的基金持仓明细" |

## 6. 数据源 (14 个采集器)

通过 opencli (浏览器桥) + Tushare/Qlib 本地库. 部分需要 Chrome cookie.

| 站点 | 采集内容 | 认证 |
|---|---|---|
| **Tushare / Qlib** | 日线行情/财务/估值 (本地二进制库) | token |
| **东方财富** | 7×24 快讯 / 龙虎榜 / 股东 | 公开 |
| **新浪财经** | 新闻 | 公开 |
| **同花顺** | 热股榜 / 问财选股 / 资金流(个股·概念·行业·大单) / 概念板块 | 公开+plugin |
| **雪球** | 单股评论 / 热门动态 / 关注时间线 / 热股榜 / 财报日历 / 实时行情 / 自选股 | cookie |
| **蛋卷基金** | 账户总览 / 持仓明细 | cookie |

数据落地: SQLite (新闻/情绪/资金流/盯盘规则) + Qlib 二进制 (行情) + Parquet (财务).

## 7. 斜杠命令

| 命令 | 作用 |
|---|---|
| `/help` | 帮助 |
| `/tools` | 列出全部工具 |
| `/reset` | 清空对话历史 |

## 8. 权限模式 (Claude-Code 风格, 3 档)

状态行常驻显示当前模式 + 不同图标颜色:

| 模式 | 图标 | 行为 |
|---|---|---|
| **default** 🛡 (青) | 默认 | instant/seconds 工具自动跑; **minutes 级**(研报/因子基准) 弹 y/n |
| **safe** 🚦 (黄) | 审计 | **每个工具调用前**都问 y/n, 逐步批准 |
| **auto** ⚡ (品红) | 放飞 | 全部自动, 不问 |

确认时输入: `y` 同意 / `n` 拒绝 / `a` 同意且本会话该工具永不再问 / `ESC` 取消整轮.
模式持久化到 `~/.financial-analyst/buddy.yaml`, 重启恢复.

## 9. 状态行信息 (常驻, 信息密度高 — 设计要做好层级)

一行内同时显示 (按需拼接):
```
🛡 default · qwen3.5-plus (qwen) · 👁 盯盘5m(交易中) · 🪙 2.3k tok (↑1800 ↓500, 4 calls) · auto-approved: news_query
```
- 权限模式 (图标+色)
- 当前 LLM (model + provider)
- 盯盘状态 + 交易时段 (交易中/午休/已收盘/休市)
- 本会话 token 累计 + 调用次数
- 已自动批准的工具列表

**设计挑战**: 这行信息量大, 终端里挤在一行. Web/桌面版可以拆成状态栏多个 chip.

## 10. 关键交互特性 (设计要支持的"动作")

| 特性 | 说明 |
|---|---|
| **边想边输入** | agent 思考时用户可继续打字, Enter 排队 (单槽, 新的替换旧的) |
| **ESC 逐层取消** | 一次 ESC 取消当前轮; 有排队的话再按取消排队 |
| **翻历史** | PageUp/PageDown / 鼠标滚轮翻看旧对话; End 回到最新 (翻历史时新内容不会把你拽走) |
| **自动跟随** | 默认对话流自动滚到底部跟随最新输出 |
| **盯盘后台** | /watch on 后, 每 N 分钟后台评估价格提醒, 触发弹 🔔 (仅交易时段, 收盘/周末不空跑) |
| **数据时效守卫** | 查到的新闻超 18h, 结果顶部插 ⚠ 提示先刷新 |
| **工具结果截断** | 长结果对话流里截断显示 (完整内容仍喂给 LLM) |

## 11. 信息形态 (设计要承载的内容类型)

1. **对话气泡**: 用户提问 / LLM 中文总结 (markdown: 标题/列表/加粗)
2. **工具调用卡**: ▶ 工具名 + 参数; ✓/✗ 结果状态
3. **数据表格**: 资金流榜/自选股/持仓 (代码·名称·价·涨跌幅·主力净额… 中文列对齐很关键)
4. **新闻列表**: 时间戳 + 标题 + 摘要 + 来源
5. **研报**: 长文 markdown (星级评级/目标价/止损/多空论证/风控)
6. **提醒卡**: 🔔 盯盘触发 (股票 + 现价 + 涨跌幅 + 盘口状态)
7. **状态指示器**: spinner / 排队 / 确认 / 历史 / 状态行

## 12. 给设计师的优先级提示

**高频核心路径** (设计重点打磨):
1. 提问 → 思考 → 总结 (基础对话流)
2. `stock_brief` 速览 (最常用的"看下 X 怎么样")
3. 盯盘 alert + 🔔 触发 (实时性, 要醒目)
4. 权限确认 y/n (安全关口, 要清晰)

**中频**: 资金流榜/新闻/研报 (信息密度高的展示)

**低频/进阶**: 因子工具 / 个人账户 / dream_review

**当前痛点 (设计可改善的)**:
- 状态行信息挤在一行 → 可拆 chip / 状态栏
- 数据表格在终端靠空格对齐 → 真表格组件更好
- 研报长文在对话流里滚动 → 可侧栏/弹层展示
- 多个状态条堆叠占垂直空间 → 可合并/浮层

## 13. 技术形态 (现状)

- **运行**: 终端命令 `financial-analyst chat`
- **UI 框架**: prompt_toolkit (Python 全屏 TUI) + Rich (markdown→ANSI)
- **LLM**: LiteLLM 多provider (默认 qwen3.5-plus, 可切 claude/gpt/deepseek)
- **配置**: `~/.financial-analyst/` (buddy.yaml 偏好 / alerts.yaml 盯盘 / news.sqlite 数据)

> 如果重做界面: 核心价值是"自然语言驱动 A 股投研 + 可打断的异步工具执行 +
> 后台盯盘". 形态可以是终端美化、桌面 app、或 Web. 关键是把 §4 状态机 和
> §11 信息形态 呈现清楚.
